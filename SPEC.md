React + Vite + TypeScript で、
「霧幻学苑セレスティアーク － ムネモシュネ・アーカイブ同期端末 －」
を実装してください。

これは一般的なソーシャルゲームのガチャUIではなく、
霧幻学苑セレスティアークの世界観に基づいた
「ムネモシュネに記録された個体ログを、学苑端末上で呼び出し・照合・再構成・同期表示するシステム」
として設計してください。

==================================================
【最重要コンセプト】
==================================================

このシステムは「排出」ではなく「同期」です。

- カード = ムネモシュネに記録された個体ログ
- ガチャ実行 = ログ呼び出し / 同期開始
- 排出 = 同期成功 / 顕現成功
- 未所持 = 未同期
- 演出動画 = 過去ログ再生記録
- 一覧 = 同期ログ一覧
- 10連 = 一括ログ復元
- スキップ = 強制同期

UI・文言・演出・コンポーネント構造も、
できる限りこの世界観に沿って設計してください。

==================================================
【世界観表現の前提】
==================================================

- ムネモシュネは記録の集積・照合・保全を担う
- グリフは記録装置・認証媒体のような役割を持つ
- ヴェイル干渉により、ノイズ、歪み、欠損、再構成演出が発生する
- SSR/UR動画は単なる挿入動画ではなく、「過去ログ再生記録」として扱う

==================================================
【技術要件】
==================================================

- React
- Vite
- TypeScript

推奨:
- Zustand もしくは React Context で状態管理
- Framer Motion でUI演出補助
- Canvas 2D で粒子演出
- HTML5 video で動画再生
- localStorage で同期状態保存

要件:
- 型安全に実装すること
- 巨大な1ファイル構成にしないこと
- 演出状態は enum または明示的 state machine 的に管理すること
- 画像 / 動画読み込み失敗時のフォールバックを入れること
- localStorage破損時も安全に復旧できるようにすること

【演出の最重要要件】
==================================================

SSR/UR動画は、単独で唐突に再生してはいけません。

必ず、
「同期システム演出の中に自然に接続されたログ再生ウィンドウ」
として見せてください。

つまり、
動画の前後にノイズ・粒子・グリッド・警告UI・同期率変動・端末フレーム変形などを入れ、
動画が単なるmp4再生ではなく
「同期演出の一部」として成立するようにしてください。

必須要件:
- 動画開始前に、同期ウィンドウが形成される
- 動画はそのウィンドウ内部で再生される
- 動画の周囲では端末UIが生きている
- 動画終了後、最終フレームや残像がカード顕現へ連続する
- 動画だけ浮かないこと
- スキップ時も短縮版の形成 → 顕現感を残すこと

==================================================
【SSR演出仕様】
==================================================

流れ:
1. 同期開始
2. ノイズ発生（記録欠損）
3. 粒子再構成
4. ログ再生フレーム形成
5. SSR動画再生
6. 動画最終フレームを短く保持
7. カード化 / 顕現
8. SYNC COMPLETE
9. 結果表示

意図:
- 破損記録の照合復元
- 派手だが制御されている
- 上品で格好いい
- URほど暴走しない

==================================================
【UR演出仕様】
==================================================

流れ:
1. 同期開始
2. ヴェイル歪み
3. グリッド線出現
4. データ再構築リング
5. 高出力ログ再生フレーム形成
6. UR動画再生
7. 同期暴走フラッシュ
8. 動画残像からカード顕現（回転あり）
9. 残留粒子
10. SYNC COMPLETE
11. 結果表示

意図:
- 通常同期を超える強度
- 危うさと神秘性
- “何か大きな記録を強引に引きずり出した” 感覚
- 世界観の核に近い重さ

【動画まわりの実装方針】
==================================================

- 動画は全画面ベタ置きにしない
- 同期ウィンドウ内で再生する
- 動画の上に薄いUIオーバーレイを重ねる
- 動画の背面または周囲で粒子・スキャンライン・警告UIを継続表示する
- 動画開始前に 0.4〜1.2秒程度の形成演出を入れる
- 動画終了後に 0.2〜0.5秒程度の最終フレーム保持を入れる
- その後 CardReveal3D にシームレス接続する

- 未同期はノイズ表示
- 同期済みは通常表示
- 詳細モーダルが開ける
